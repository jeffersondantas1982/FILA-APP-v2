01. Introdução

Visão Geral

O FILA-APP é uma solução tecnológica avançada desenvolvida para gerenciar fluxos de atendimento em ambientes de alta demanda, como Unidades de Saúde e Postos de Atendimento. O sistema integra inteligência de dados, real-time communication e acessibilidade para garantir que o tempo do paciente e do profissional seja respeitado e otimizado.

Principais Diferenciais

  • Tempo Real: Chamadas instantâneas via WebSockets.
  • Acessibilidade por Voz: Chamada de senhas audível para inclusão total.
  • Transparência: Painéis públicos e acompanhamento de fila.
  • Gestão por Dados: Mapas de calor e rankings de produtividade.

Interface de Acesso Segura e Minimalista

Requisitos Mínimos

Para o funcionamento estável, recomenda-se:

  • Servidor: Node.js 16+ e MySQL 8.0+.
  • Recursos: 2GB de RAM e 1GHz de CPU.
  • Rede: Porta 3000 liberada para tráfego TCP na rede local.

Fluxo de Implantação

1. Clone o repositório ou descompacte os arquivos na pasta do servidor.

2. Configuração do Banco: Crie o schema fila_app e execute o arquivo database.sql.

3. Variáveis de Ambiente: Configure o arquivo .env com os dados do MySQL e uma chave JWT única.

DB_HOST=localhost
DB_USER=root
DB_PASS=sua_senha
DB_NAME=fila_app
PORT=3000
JWT_SECRET=sua_chave_secreta

4. Execução: Use o script npm start ou 3-START.bat.

04. Módulo de Operação (Guichês)

A ferramenta principal do profissional de saúde. Focada em agilidade e precisão no atendimento.

Console de Operação: Fluxo fluido de chamadas

Ciclo de Atendimento:

  1. Assumir Setor: Escolha o setor em que você está operando para carregar a fila específica.
  2. Chamar Próximo: O sistema utiliza algoritmos de prioridade para chamar senhas Preferencialmente antes das normais.
  3. Chamar Novamente: Utilize o botão de rechamada caso o paciente não compareça de imediato.
  4. Iniciar/Concluir: Marque o tempo exato do atendimento para KPIs de performance e produtividade.
  5. Redirecionar: Envie o paciente para Triagem -> Médico -> Exames mantendo a mesma senha no sistema.

Alertas e Produtividade:

  • Notificações Nativas: Ative o botão para permitir que o Chrome envie notificações de sistema, mesmo com o navegador minimizado.
  • Tab Flashing: Quando o operador está em outra aba, o título do navegador pisca alternadamente entre "🔔 NOVA SENHA!" e o título original para atrair atenção visual.
  • Som de Alerta: Um sinal sonoro de alta frequência é emitido a cada nova entrada na fila do setor ativo.
  • Modo Mudo: É possível silenciar os alertas caso o operador esteja em um ambiente que exija silêncio absoluto.

05. Terminais e Displays Públicos

1. Totem de Triagem

O ponto de triagem operado pelo Porteiro ou Recepcionista para recepção e classificação inicial do fluxo.

Interface de Toque: Agilidade na escolha entre Atendimento Normal ou Preferencial

Lógica de Emissão:
  • Normal: Gera senhas sequenciais (Ex: A01, A02) para o público geral.
  • Preferencial: Utiliza prefixo distinto e prioridade máxima na fila do operador, seguindo a legislação de acessibilidade.

2. Painel de TV (Display)

O centro informativo da sala de espera, combinando áudio e vídeo.

Sintetizador de Voz: "Senha B25, dirija-se à Sala 04"

Tecnologia de Voz (TTS): O sistema utiliza síntese de voz nativa do navegador para garantir que o áudio seja claro e customizável sem dependência de internet.

Indicadores de Conexão: No canto inferior direito, um LED virtual monitora o WebSocket:
  • Verde: Conectado e recebendo dados em tempo real.
  • Laranja: Tentando restabelecer conexão com o servidor.
  • Vermelho: Desconectado. Verifique a rede ou o status do serviço Node.js.

06. Governança e Administração

1. Configuração da Identidade da Unidade

O primeiro passo após a instalação é personalizar o sistema com o nome do hospital ou clínica.

Passo a Passo:
  1. Acesse o menu Configurações no menu lateral.
  2. No card "Configurações Gerais", localize o campo Nome da Instituição.
  3. Digite o nome oficial (Ex: Unidade de Saúde X).
  4. Clique em Salvar Alterações. Este nome aparecerá instantaneamente no topo de todas as telas e painéis.

2. Configuração de Impressão de Senhas

O sistema permite operar com ou sem o uso de impressoras de cupom (térmicas).

Como Ativar/Desativar:
  • Vá em Configurações.
  • Localize o interruptor (toggle) Impressão de Senhas.
  • Ligado: Ao gerar uma senha (no Totem ou Recepção), a janela de impressão do navegador será aberta automaticamente configurada para cupom 80mm/58mm.
  • Desligado: O sistema apenas exibe a senha na tela para entrega manual pelo recepcionista.

3. Gerenciamento de Setores

Os setores definem para onde os pacientes serão enviados e qual guichê será responsável pela chamada.

Como Cadastrar um Novo Setor:
  • Clique em Novo Setor na tela de gerenciamento de setores.
  • Nome: Ex: "Consultório Médico" ou "Triagem".
  • Sala/Guichê: Identificação física do local.
  • Prefixo: Letra que precede a senha (Ex: 'M' para Médico -> M01, M02...).

Painel de Controle: Estruturação de Fluxo por Setor

5. Matriz de Perfis e Permissões

O sistema adota o modelo RBAC (Role-Based Access Control) para garantir que cada usuário visualize apenas o necessário.

Perfil Nível de Acesso Principais Responsabilidades
SUPER_ADMIN Acesso Total Configuração do servidor, exclusão de dados e gestão integral de usuários e setores.
GERENTE Gerencial Gestão de setores, visualização de relatórios, Dashboards e reset diário de senhas.
OPERADOR Operacional Chamada de senhas, redirecionamento e finalização de atendimentos no guichê.
RECEPCIONISTA Entrada Emissão manual de senhas na recepção para pacientes que não usam o totem.
PORTEIRO Entrada Emissão de senhas básicas e orientação de fluxo na chegada da unidade.

5.1 Configuração de Impressão Silenciosa (Direct Print)

Para que a senha seja impressa automaticamente sem abrir a janela de confirmação do navegador, siga estas etapas:

O recurso de "Impressão Silenciosa" (Kiosk Printing) é uma configuração do navegador (Chrome/Edge) para ambientes profissionais.
Como configurar no Windows:
  1. Certifique-se que sua impressora térmica está configurada como a Impressora Padrão do Windows.
  2. Crie um atalho do Google Chrome na sua área de trabalho.
  3. Clique com o botão direito no atalho e vá em Propriedades.
  4. No campo Destino, adicione o seguinte comando ao final (após as aspas):
    --kiosk-printing
  5. Clique em OK e abra o sistema através deste novo atalho.

6. Relatórios e Auditoria

O módulo de relatórios permite auditar cada segundo da operação. Utilize os filtros laterais para refinar sua busca.

Recursos Extras do Módulo:
  • Filtros Dinâmicos: Filtre por período, setor, modalidade (Normal/Prioridade) e status (Concluído/Cancelado).
  • Exportação Direta: Gere arquivos CSV compatíveis com Excel e PowerBI.
  • Tradução Automática: O sistema converte status técnicos (ex: `not_shown`) para termos amigáveis (ex: `Não Compareceu`).

Exportação Direta: Dados prontos para Excel

7. Zona de Perigo (Manutenção do Sistema)

Local de acesso restrito para limpeza técnica da base de dados.

Operações de Segurança:
  • Reset Diário: Zera o contador de senhas (ex: de B99 para B01).
  • Zerar Banco: Remove dados de teste antes de entrar em produção.
  • Demo Loader: Popula o sistema com dados fictícios para treinamento.

Controle Crítico: Apenas para Super Administradores

07. Sistema e Manutenção

Automação (.bat)

  • 1-INSTALAR.bat: Prepara o ambiente Node.js.
  • 2-INICIALIZAR_BANCO.bat: Cria o banco MySQL.
  • 3-START.bat: Inicia a aplicação.

Variáveis de Ambiente (.env)

O arquivo .env centraliza as configurações sensíveis. Nunca compartilhe este arquivo publicamente.

DB_HOST=localhost      # Servidor MySQL
DB_NAME=fila_app       # Nome do Banco
PORT=3000              # Porta de acesso Web

Troubleshooting

Caso o Painel de TV não exiba chamadas, verifique se o Script 3 está rodando e se há conexão de rede entre a TV e o Servidor.
